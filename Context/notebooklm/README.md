# NotebookLM — Organização

Crie **um notebook por tópico**. Cada arquivo nesta pasta contém sugestões de fontes (links, vídeos, PDFs) e perguntas pra fazer ao notebook.

## Notebooks sugeridos

| Notebook | Arquivo de referência | Foco |
|----------|----------------------|------|
| 📓 TDD + Testing Library | `01-tdd-testing.md` | Testes, Vitest, React Testing Library |
| 📓 SQL + PostgreSQL | `02-sql-postgres.md` | SQL, modelagem relacional, JPA |
| 📓 Estruturas de Dados + LeetCode | `03-estruturas-dados.md` | Arrays, Maps, Big O, algoritmos |
| 📓 Java + Spring Boot | `04-java-spring.md` | Spring, JPA, conceitos de framework |

## Como usar

1. Abra o NotebookLM
2. Crie um novo notebook com o nome do tópico
3. Adicione as fontes sugeridas no arquivo correspondente (links, PDFs, YouTube, texto colado)
4. Use o chat pra fazer as perguntas listadas
5. Ative o "Guia" pra gerar um resumo automático do notebook

## Regras

- **Máximo de 50 fontes por notebook** (limite do NotebookLM)
- **Não misture tópicos** — cada notebook é um assunto
- **Archive** notebooks antigos em vez de deletar
- **Use o chat pra perguntar**, não pra guardar anotações (as anotações ficam nos .md daqui)
